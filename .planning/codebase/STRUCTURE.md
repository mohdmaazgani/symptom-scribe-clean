## Directory Purposes

### `src/pages/` — Route Page Components
- **Purpose:** One file per route/screen. Pages are self-contained, each managing its own data fetching and state.
- **Contains:** 22 page `.tsx` files — 10 protected app pages, 7 public legal/content pages, 2 auth pages, 1 landing page, 1 404 page, 1 blog detail page
- **Key files:**
  - `Index.tsx` (613 lines) — Landing page with Hero, features, FAQ, testimonials, footer
  - `Auth.tsx` (317 lines) — Login/signup with Supabase auth, password reset
  - `Dashboard.tsx` (323 lines) — Stats cards, wellness gauge, recent symptom list
  - `Emergency.tsx` (1064+ lines) — Largest page: emergency numbers, first aid guides, crisis hotlines, hospital finder, SMS broadcast, panic button
  - `BrainGames.tsx` (1862 lines) — Games like memory match, reaction time, sequence recall, math challenge
  - `AIHealthAssistant.tsx` (487 lines) — Voice-enabled symptom chat with speech recognition
  - `Settings.tsx` (370 lines) — Password change, account deletion
  - `Metrics.tsx` (701 lines) — Health metric tracking with chart (recharts), offline sync
  - `History.tsx` (402 lines) — Symptom history with search, filter, CSV export, offline sync

### `src/components/` — Reusable Components
- **Purpose:** Shared UI building blocks used across pages
- **Contains:** ~55 component files
- **Subdirectories:**
  - **`src/components/ui/`** — 50+ shadcn/ui primitives (Button, Card, Dialog, Input, Select, Table, Tabs, Sheet, Sidebar, etc.). These are auto-generated/styled Radix UI wrappers following shadcn conventions. Each is a single `.tsx` or `.ts` file with Radix + `cn()` styling.
  - **`src/components/legal/`** — `LegalPageLayout.tsx` (shared layout for legal pages with sticky header, icon hero), `Section.tsx` (card section wrapper), `PageFooter.tsx` (minimal footer with legal links)
  - **`src/components/registration/`** — `MultiStepSignUp.tsx` (466 lines) — 5-step signup flow (Account → Personal → Health → Emergency → Review) with Supabase auth registration + profile upsert

### `src/lib/` — Utilities and Business Logic
- **Purpose:** Shared logic, storage abstractions, encryption, API helpers. No JSX here — pure TypeScript.
- **Contains:** 8 files serving as the "data layer" of the application

### `src/hooks/` — Custom React Hooks
- **Purpose:** Encapsulated stateful logic shared across components
- **Contains:** 3 hooks

### `src/integrations/` — Third-Party Service Clients
- **Purpose:** Provider-specific client initialization and types
- **Contains:** Supabase client + auto-generated types

### `src/data/` — Static Data
- **Contains:** `blogData.ts` — blog post content array with full article text, metadata, prev/next navigation

### `src/test/` — Test Infrastructure
- **Purpose:** Test utilities and global setup for Vitest + React Testing Library
- **Key detail:** `AllProviders.tsx` wraps tests with `QueryClientProvider` + `MemoryRouter`, creating a fresh `QueryClient` for each test to avoid cache pollution

### `supabase/migrations/` — Database Migrations
- **Purpose:** Sequential SQL migrations for Supabase PostgreSQL
- **Files (chronological):**
  - `20251028170142_...sql` — Initial schema setup
  - `20260522153000_add_cascade_deletes.sql` — Cascade delete foreign keys
  - `20260523173000_create_profiles_trigger.sql` — Auto-create profile on user signup
  - `20260604000000_add_xp_and_level_to_profiles.sql` — Gamification fields
  - `20260608000000_add_foreign_keys_to_user_tables.sql` — FK constraints
  - `20260609144200_enforce_rls_policies.sql` — Row-level security
  - `20260612100000_secure_gamification.sql` — Gamification security

### `public/` — Static Assets
- **Purpose:** Files served as-is by Vite, including PWA icons, favicon, Netlify redirects, and robots.txt

## Naming Conventions

**Files:**
- **React components:** PascalCase `.tsx` — `ChatInterface.tsx`, `PasswordStrengthMeter.tsx`, `LegalPageLayout.tsx`
- **Hooks:** camelCase prefixed with `use` — `useMetricsHistory.ts`, `use-toast.ts`, `use-mobile.tsx`
- **Utilities:** camelCase `.ts` — `env.ts`, `encryption.ts`, `storage.ts`, `utils.ts`
- **Data files:** camelCase `.ts` — `blogData.ts`
- **Test files:** `{name}.test.tsx` or `{name}.test.ts` co-located next to source — `PasswordStrengthMeter.test.tsx`, `storage.test.ts`, `password-strength.test.ts`
- **Config files:** kebab-case — `vite.config.ts`, `tailwind.config.ts`, `postcss.config.js`, `eslint.config.js`
- **UI primitives:** kebab-case `.tsx` — `alert-dialog.tsx`, `dropdown-menu.tsx`, `navigation-menu.tsx`

**Directories:**
- **All lowercase** — `components/`, `hooks/`, `lib/`, `pages/`, `integrations/`, `ui/`, `legal/`, `registration/`, `test/`

**Code Elements:**
- **Exported functions/components:** Named exports for utility components (`export function AnimatedThemeToggler`), default exports for pages (`const Auth = () => ...; export default Auth`)
- **Interfaces:** PascalCase — `Props`, `State`, `Message`, `SymptomEntry`, `Stats`, `HealthFact`
- **Types:** PascalCase with `Type` suffix where needed — `StorageKey`, `CachedTable`
- **Constants:** UPPER_SNAKE_CASE — `MOBILE_BREAKPOINT`, `MAX_CHARS`, `TOAST_LIMIT`, `STORAGE_KEYS`
- **CSS classes:** Tailwind utility classes throughout (no custom CSS modules)
- **State setters:** `set{StateName}` — `setMessages`, `setLoading`, `setActiveSessionId`

## Where to Add New Code

### New Feature Page (protected)
1. Create page component in `src/pages/{FeatureName}.tsx`
2. Add route in `src/App.tsx` wrapped in `<ProtectedRoute><Layout>...</Layout></ProtectedRoute>`
3. Register navigation link in `src/components/AppSidebar.tsx` `menuItems` array

### New Feature Page (public)
1. Create page component in `src/pages/{FeatureName}.tsx`
2. Add route in `src/App.tsx` without ProtectedRoute/Layout wrapper

### New Component
- **Reusable UI primitive:** Add to `src/components/ui/{name}.tsx` following shadcn pattern
- **Feature-specific component:** Add to `src/components/{FeatureName}.tsx`
- **Legal page component:** Add to `src/components/legal/{Name}.tsx`

### New Hook
- Add to `src/hooks/{useName}.ts` (or `.tsx` if it renders JSX)

### New Utility
- Add to `src/lib/{name}.ts`

### New Data File
- Add to `src/data/{name}.ts`

### New Supabase Edge Function
- Add to `supabase/functions/{function-name}/index.ts`
- Shared utilities go in `supabase/functions/_shared/`

### New Test
- Co-locate: `src/{folder}/{filename}.test.tsx` next to the source file
- Use `@/test/utils` render helper for components that need providers

### New Database Migration
- Add file to `supabase/migrations/{timestamp}_{description}.sql`
